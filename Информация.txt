Обязательно нужно:
+ 1. Использование архитектуры Model-View-Controller или другой, обеспечивающей аналогичное разделение.

+ 2. Интерфейс должен быть простым в использовании и интуитивно понятным для пользователя.

+ 3. Требуется динамическое обновление интерфейса.

+ 4. Интерфейс должен быть основан на текстовом пользовательском интерфейсе (Text-based UI),
а не на простом выводе сообщений с помощью print.

- 5. Документация проекта (минимум README с описанием функциональности).

+- 6. Качество кода (соответствующее именование классов, методов, переменных).

+ 7. Корректное моделирование логики в виде отдельных классов/методов/функций.

+ 8. Написание модульных тестов (обязательно) и интеграционных (по желанию).

+ 9. Использование инструмента для управления зависимостями и сборки проекта (например, Maven/Gradle).

- 10. Использование системы контроля версий Git (дополнительно оценивается).

+ 11. Система управления и отрисовки ASCII-артов. (Сложно сделать)
